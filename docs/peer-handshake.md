# TCP Peer Handshake

## Overview

To communicate with a peer in the BitTorrent protocol, you must first establish a **TCP connection** and perform a **handshake**. The handshake is the very first message exchanged — both sides send and receive one in the same format. After a successful handshake, the connection transitions to the peer wire protocol's length-prefixed message stream.

---

## Peer Protocol Background

- Operates over **TCP** (or uTP).
- Connections are **symmetrical** — messages look the same in both directions.
- Each end of a connection has two bits of state: **choked/unchoked** and **interested/not interested**.
- Connections start out **choked** and **not interested**.
- The protocol begins with a handshake, followed by a never-ending stream of **length-prefixed messages**.

---

## Handshake Message Format

The handshake is a **68-byte** message composed of the following parts, sent in order:

| Field                  | Size     | Description                                                               |
| ---------------------- | -------- | ------------------------------------------------------------------------- |
| Protocol string length | 1 byte   | Always `19` (decimal) — the length of `"BitTorrent protocol"`             |
| Protocol string        | 19 bytes | The ASCII string `BitTorrent protocol`                                    |
| Reserved bytes         | 8 bytes  | All zeros (`0x00`). Used for protocol extensions.                         |
| Info hash              | 20 bytes | The raw SHA-1 hash of the bencoded `info` dictionary (NOT the hex string) |
| Peer ID                | 20 bytes | A unique identifier for this client (20 random bytes)                     |

**Total: 1 + 19 + 8 + 20 + 20 = 68 bytes**

### Important Notes

- The **info hash** is the raw 20-byte SHA-1 digest, **not** the 40-character hexadecimal representation.
- The **peer ID** should be 20 randomly generated bytes. In practice, clients often use a convention like `-XX1234-<random>`, but raw random bytes work.
- If the info hashes from both sides don't match, the connection is **severed**.

## Step-by-Step Implementation (Go)

### 1. Parse the Torrent File

Read and decode the `.torrent` file to extract:

- The `info` dictionary (as raw bencoded bytes for hashing).
- The tracker URL and peer list (from a prior `peers` command or tracker response).

You already have this logic in the `info` and `peers` commands.

### 2. Establish a TCP Connection

Use Go's `net.Dial` to connect to the peer's `<ip>:<port>`:

```go
conn, err := net.Dial("tcp", peerAddress) // e.g. "178.62.85.20:51489"
if err != nil {
    fmt.Fprintln(os.Stderr, "Error connecting:", err)
    os.Exit(1)
}
defer conn.Close()
```

### 3. Build the Handshake Message

Construct the 68-byte handshake buffer:

```go
var handshake [68]byte

// 1) Protocol string length (1 byte)
handshake[0] = 19

// 2) Protocol string (19 bytes)
copy(handshake[1:20], []byte("BitTorrent protocol"))

// 3) Reserved bytes (8 bytes) — already zero-valued

// 4) Info hash (20 bytes) — raw SHA-1, not hex
copy(handshake[28:48], infoHash[:])

// 5) Peer ID (20 bytes) — random
var peerId [20]byte
_, err = rand.Read(peerId[:])
if err != nil {
    fmt.Fprintln(os.Stderr, "Error generating peer ID:", err)
    os.Exit(1)
}
copy(handshake[48:68], peerId[:])
```

> **Remember to import `"crypto/rand"`** for generating random bytes.

### 4. Send the Handshake

Write the entire 68-byte message to the TCP connection:

```go
_, err = conn.Write(handshake[:])
if err != nil {
    fmt.Fprintln(os.Stderr, "Error sending handshake:", err)
    os.Exit(1)
}
```

### 5. Receive the Peer's Handshake

Read exactly 68 bytes back from the connection:

```go
var response [68]byte
_, err = io.ReadFull(conn, response[:])
if err != nil {
    fmt.Fprintln(os.Stderr, "Error reading handshake:", err)
    os.Exit(1)
}
```

> Use `io.ReadFull` instead of `conn.Read` to guarantee you receive all 68 bytes.

### 6. Extract and Print the Peer ID

The peer's ID is the last 20 bytes (offset 48–67) of the response. Print it as a hexadecimal string:

```go
receivedPeerId := response[48:68]
fmt.Println("Peer ID:", hex.EncodeToString(receivedPeerId))
```

### Expected Output

```
Peer ID: 0102030405060708090a0b0c0d0e0f1011121314
```

_(The actual hex value will vary depending on the peer.)_

## Complete `handshake` Command Example

```go
case "handshake":
    if len(os.Args) < 4 {
        fmt.Fprintln(os.Stderr, "Usage: ./runner.sh handshake <torrent_file> <peer_address>")
        os.Exit(1)
    }

    torrentFile := os.Args[2]
    peerAddress := os.Args[3]

    // --- Parse torrent file ---
    data, err := os.ReadFile(torrentFile)
    if err != nil {
        fmt.Fprintln(os.Stderr, "Error:", err)
        os.Exit(1)
    }

    dataStr := string(data)
    pos := 0
    _, rawValues, err := decodeDict(dataStr, &pos)
    if err != nil {
        fmt.Fprintln(os.Stderr, "Error:", err)
        os.Exit(1)
    }

    infoHash := sha1.Sum([]byte(rawValues["info"]))

    // --- Connect to peer ---
    conn, err := net.Dial("tcp", peerAddress)
    if err != nil {
        fmt.Fprintln(os.Stderr, "Error connecting:", err)
        os.Exit(1)
    }
    defer conn.Close()

    // --- Build handshake ---
    var handshake [68]byte
    handshake[0] = 19
    copy(handshake[1:20], []byte("BitTorrent protocol"))
    // reserved bytes [20:28] are already zero
    copy(handshake[28:48], infoHash[:])

    var peerId [20]byte
    _, err = rand.Read(peerId[:])
    if err != nil {
        fmt.Fprintln(os.Stderr, "Error generating peer ID:", err)
        os.Exit(1)
    }
    copy(handshake[48:68], peerId[:])

    // --- Send handshake ---
    _, err = conn.Write(handshake[:])
    if err != nil {
        fmt.Fprintln(os.Stderr, "Error sending handshake:", err)
        os.Exit(1)
    }

    // --- Receive handshake ---
    var response [68]byte
    _, err = io.ReadFull(conn, response[:])
    if err != nil {
        fmt.Fprintln(os.Stderr, "Error reading handshake:", err)
        os.Exit(1)
    }

    // --- Print peer ID ---
    receivedPeerId := response[48:68]
    fmt.Println("Peer ID:", hex.EncodeToString(receivedPeerId))
```

> **New import needed:** `"crypto/rand"` (in addition to the existing `"crypto/sha1"`).

## Quick Reference: Byte Offsets

```
Offset  0:       protocol string length (0x13 = 19)
Offset  1–19:    "BitTorrent protocol"
Offset 20–27:    reserved (all zeros)
Offset 28–47:    info hash (20 bytes, raw SHA-1)
Offset 48–67:    peer ID (20 bytes)
```
